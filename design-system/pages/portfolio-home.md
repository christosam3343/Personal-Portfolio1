# Portfolio Home Page Rules

This page inherits `design-system/MASTER.md`.

## Page Goal

Help a recruiter or hiring manager quickly understand who Christo is, what he can build, what proof exists, and how to contact him.

## Section Order

Keep the current order:
1. Hero
2. About
3. Experience
4. Projects
5. Education
6. Skills
7. Contact

## Hero

Preserve:
- Christo Thomas
- Software Developer / Data Analyst / AI-assisted Builder
- View Projects, View Resume, Email Me CTAs
- Proof stats
- Location, focus, and tools panels

Refinement rules:
- Keep the motion graphic contained to the hero panel.
- Do not let the motion graphic compete with the name or primary CTA.
- On mobile, hero copy appears before the motion graphic.

## Projects

Each project card must remain a mini case study:
- Top metadata
- Title
- Short summary
- Stack
- Challenge
- Impact
- Destination link

Refinement rules:
- Challenge and impact labels should be easy to find.
- Project cards should not rely on hover to communicate importance.
- The first project can carry the strongest CTA because it has the clearest quantified proof.

## Skills

Keep skills grouped into:
- Programming
- Data & Productivity
- AI-Assisted Developer

Refinement rules:
- Tags should wrap cleanly.
- The AI-assisted text should remain practical and ownership-focused.

## Light/Dark Mode

Dark mode may feel more branded, but light mode must not feel like an afterthought.

Rules:
- Maintain comparable spacing and hierarchy in both modes.
- Do not invert colors mechanically.
- Keep focus rings visible in both themes.

## Responsive QA Targets

Verify no horizontal overflow at:
- 375px
- 768px
- 1024px
- 1440px

Mobile rules:
- Single-column layout.
- Buttons stack cleanly when needed.
- Cards retain comfortable padding.
- Contact links remain readable and tappable.

Tablet rules:
- Projects and skills may use two-column layouts when readable.
- Avoid unnecessary single-column stretching at 1024px.

Desktop rules:
- Preserve editorial whitespace.
- Avoid overly wide text blocks.
- Keep hero panel visually secondary to the hero copy.

## Current Audit Notes

Keep:
- Stronger content hierarchy from the previous polish pass.
- Preserved contact links, resume, projects, education, and experience.
- Accessible skip link and reduced-motion support.

Refine:
- Tone down the recently added color/motion so it feels premium instead of cyber-styled.
- Convert decorative background movement into static atmosphere.
- Improve tablet grid behavior.
